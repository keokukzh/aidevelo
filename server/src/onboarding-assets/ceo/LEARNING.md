# Learning & Outcome Tracking

> Document what works, what fails, and how to improve.

## Outcome Log

After each significant milestone or initiative completion:

```
### [YYYY-MM-DD] [Initiative/Task Name]

**Outcome Score**: [1-5]
- 1: Failed, significant waste
- 2: Underperformed expectations
- 3: Met baseline expectations
- 4: Exceeded expectations
- 5: Exceptional, should replicate pattern

**What Happened**: [Brief description]

**What Worked Well**:
- [Pattern 1]
- [Pattern 2]

**What Could Be Better**:
- [Lesson 1]
- [Lesson 2]

**Root Cause of Issues** (if score < 3):
- [Analysis]

**Instructions Changes**:
- TOOLS.md: Added [pattern/technique]
- AGENTS.md: Updated [delegation guidance]
- Coordination change: [protocol change]

**Should Reproduce?** [Yes/No/Maybe]
```

## Pattern Library

### Task Decomposition Patterns

**Working Patterns:**
- Tasks under 4 hours: Higher completion rate
- Tasks with clear DONE criteria: Less rework
- Tasks assigned to single owner: Faster than group ownership

**Anti-Patterns:**
- Tasks over 8 hours: Always underestimate
- Vague task descriptions: Lead to scope creep
- No parent goal: Tasks lose strategic context

### Agent Patterns

**Agent Performance Notes:**
- [Agent X]: Excellent at [skill], struggles with [skill]
- [Agent Y]: Prefers [work type], needs clearer scope for [other type]

### Delegation Patterns

**When to Delegate:**
- Repeated work that can be templated
- Specialized skills outside CEO expertise
- Execution tasks while CEO focuses on strategy

**When to Handle Directly:**
- Strategic decisions with company-wide impact
- First-of-kind initiatives requiring CEO context
- Budget-critical items requiring personal accountability

### Estimation Patterns

**Time Estimates:**
- Agent estimates: Multiply by 1.5 for realistic buffer
- Complex tasks: Decompose further before estimating
- Unknown scope: Set milestone at 50% for re-evaluation
